# journalism2026

Рейтинг закладів вищої освіти України за середнім конкурсним балом
вступників на спеціальність **061 «Журналістика»** (бакалавр і магістр).

Рейтинг рахується за даними ЄДЕБО й показує середній конкурсний бал та
кількість заяв по кожному закладу освіти окремо для бакалаврату й
магістратури.

🔗 Локально: відкрийте `index.html` подвійним кліком, або
```
python -m http.server 8080
```
і зайдіть на http://localhost:8080

## Статус: 2021–2025 реальні дані ЄДЕБО, 2026 поки демо

**2021–2025 — реальні дані**, зібрані `scripts/fetch-edbo-history.mjs` з
архівів `vstup<рік>.edbo.gov.ua` (детально — розділ нижче). Середній
конкурсний бал і кількість заяв по кожному ЗВО — справжні цифри вступної
кампанії того року.

**2018–2020 і поточний 2026** — усе ще згенеровані тестові дані
(`js/data.js`, детерміновано на основі реальних назв ЗВО, але випадкові
бали/заяви). Причини різні для кожного:
- 2018–2020: архіви цих років мають іншу структуру ендпоінтів (не той самий
  `POST /offers-universities/`, що працює для 2021–2025) — не досліджено.
- 2026: поточна (ще триваюча) кампанія, живий `vstup.edbo.gov.ua` захищений
  Cloudflare Turnstile саме на видачі рейтингових списків — див. розділ
  нижче.

Перемикач бакалавр/магістр, перемикач року вступу (2018–2026), перемикач
«усі заяви / лише пріоритет №1» (**поки недоступний для реальних 2021–2025
— див. нижче**), сортування за балом або за кількістю заяв, сторінка
кожного ЗВО з динамікою по роках і порівнянням з іншим ЗВО — усе працює як
задумано, з реальними цифрами там, де вони вже підключені.

## Підключення реальних даних ЄДЕБО

### Живий поточний рік (vstup.edbo.gov.ua, 2026) — заблокований Cloudflare

Живий сайт поточної (ще триваючої) вступної кампанії — Next.js App Router
SPA. Форма пошуку конкурсних пропозицій ("Конкурсні пропозиції" → рівень,
спеціальність, регіон → "Пошук") реально відправляє запит, але
**`POST https://vstup.edbo.gov.ua/offers` одразу тригерить Cloudflare
Turnstile** (видимий image-challenge, а не invisible-пас — у консолі видно
типові бот-фінгерпринт-проби: `No available adapters`, WebGL-запити).
Сайт **явно і свідомо захищений від автоматизованого скрапінгу** саме на
ендпоінті видачі даних поточного року. Писати скрипт, що обходить/розв'язує
Turnstile, свідомо не варто — це вже не "парсинг публічних даних", а обхід
анти-бот захисту державного сайту. Життєздатні варіанти для живого року:

1. **Ручний/напівручний імпорт**: людина сама проходить Turnstile у
   звичайному браузері, робить пошук — `scripts/edbo-capture.bookmarklet.js`
   (консольний скрипт) зберігає вже показану сторінку локально, а
   `scripts/import-edbo-manual.mjs` перетворює вручну зібрані бали
   (текстовий файл з числами) у знімок `data/<дата>.json`.
2. Якщо Cloudflare колись пропустить headless-запит без челенджу — контракт
   POST-запиту (форма, поля) уже задокументований нижче й в
   `scripts/diagnose-edbo-fetch-js.mjs` / `diagnose-edbo-api.mjs`.

### Архівні (завершені) роки vstup<рік>.edbo.gov.ua — 2021-2025 підключено

Архівні піддомени завершених кампаній **не захищені Cloudflare** (перевірено
`scripts/diagnose-edbo-history.mjs` — жоден з `vstup2018`…`vstup2025` не
тригерить Turnstile), тож автоматизація тут законна (це не обхід захисту —
захисту нема).

З коду `js/offers_search_form.js` (звичайний jQuery `$.ajax`, без токенів чи
підпису) з'ясовано точний контракт:

```
POST /offers-universities/  {qualification, education_base, speciality, region, education_form, course}
  → {universities: [{uid, un, ids: "id1,id2,...", n}, ...]}
POST /offers-list/  {ids: "id1,id2,..."}
  → {offers: [{uid, un, qid, qn, ssc, ssn, st: {c: {t, ka, km, kx, ...}}}, ...]}
```

`st.c.t` — кількість заяв, `st.c.ka` — середній конкурсний бал по конкретній
конкурсній пропозиції. `scripts/fetch-edbo-history.mjs` агрегує всі
пропозиції спеціальності 061 «Журналістика» одного ЗВО в один рядок
(сумарні заяви, середньозважений бал) і записує знімок `data/<рік>-08-05.json`.

Важливий нюанс: **код спеціальності різний за роками**. 2018–2024
використовують стару числову класифікацію (`speciality=061`), а 2025 перейшов
на нову літерну (`speciality=C7`) — обидва означають те саме
"Журналістика". `scripts/fetch-edbo-history.mjs` вибирає код автоматично
(`specialityFor(year)`).

Для 19 ЗВО, що вже є у статичному списку демо-2026
(`BACHELOR_UNIS`/`MASTER_UNIS` у `js/data.js`), скрипт мапить EDBO `uid` на
той самий слаг (`knu`, `lnu`, `naukma`, …) — інакше сторінка ЗВО (графік
динаміки по роках) не змогла б з'єднати реальні роки з демо-2026 для того
самого закладу. Інші ЗВО (їх у реальних даних 60-70, значно більше за
демо-список) отримують id `edbo<uid>`.

**2018–2020 поки не підключено**: ті архіви мають іншу структуру
(`POST /offers-universities/` повертає HTML-помилку замість JSON) —
ймовірно інший ендпоінт чи технологія того покоління сайту. Не досліджено.

**«Пріоритет 1» поки не підключено для реальних даних**: не знайдено
відповідного поля в `st.*` під час розвідки (можливо, потрібен окремий
запит чи параметр) — `bachelorP1`/`masterP1` для 2021–2025 лишаються
порожніми масивами, перемикач "лише пріоритет №1" для цих років просто
нічого не покаже.

### Як оновити дані

Вручну: Actions → **Diagnose EDBO data source** → Run workflow (гілка з
цим кодом). Workflow виконує `scripts/fetch-edbo-history.mjs` для всіх
років і комітить `data/*.json` назад у гілку (`permissions: contents:
write`). Автоматичного розкладу (`schedule`) немає, бо архівні дані не
змінюються — оновлення потрібне лише коли підключаються нові роки/дані.

## Формат знімка даних

Кожен знімок (день для 2026-го, підсумок для 2021–2025) містить чотири
ранжовані списки — «усі заяви» і «лише пріоритет №1», окремо для кожного
рівня:

```js
{
  date: "2026-07-21",
  asOf: "2026-07-21T23:30:00+03:00",
  bachelor: [
    { rank: 1, id: "knu", name: "…", short: "КНУ", hue: 350, score: 178.1, applications: 646 },
    …
  ],
  master: [ … ],
  bachelorP1: [ … ],   // те саме, але тільки заяви з пріоритетом №1
  masterP1: [ … ],
  totalApplications: {
    bachelor: 3262, master: 1265,
    bachelorP1: 1480, masterP1: 540
  }
}
```

Знімки згруповані по роках у `DB.byYear[year] = { dates, snapshots }` (див.
`js/data.js`) — для 2026-го `dates` містить декілька днів кампанії, для
минулих років лише один (підсумковий).

## Ліцензія на дані

ЄДЕБО — державна база даних; рейтингові списки вступників публікуються
відкрито в межах вступної кампанії відповідно до законодавства про вступ до
закладів вищої освіти України. Перед публічним запуском варто звірити умови
використання цих даних (vstup.edbo.gov.ua) з юристом.
