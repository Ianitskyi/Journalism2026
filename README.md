# journalism2026

Рейтинг закладів вищої освіти України за середнім конкурсним балом
вступників на спеціальність **061 «Журналістика»** (бакалавр і магістр).

Рейтинг рахується за даними ЄДЕБО й показує середній конкурсний бал та
кількість заяв по кожному закладу освіти окремо для бакалаврату й
магістратури.

🔗 Локально: відкрийте `index.html` подвійним кліком, або
```
python -m http.server 8080
```
і зайдіть на http://localhost:8080

## Статус: демо-дані

**Зараз сайт показує згенеровані тестові дані** (`js/data.js`), а не реальний
фід ЄДЕБО. Список закладів освіти — справжні українські університети, що
готують журналістів, але бали й кількість заяв для кожного з них — випадкові,
детерміновано згенеровані для демонстрації дизайну та інтерфейсу: перемикач
бакалавр/магістр, перемикач року вступу (2021–2026), перемикач «усі заяви /
лише пріоритет №1», сортування за балом або за кількістю заяв, сторінка
кожного ЗВО з динамікою по роках і порівнянням з іншим ЗВО.

Це стосується й **історичних років** (2021–2025): дані по них так само
згенеровані, а не взяті з архіву ЄДЕБО. Для поточного 2026-го є щоденна
історія (кампанія ще триває), для минулих років — по одному підсумковому
знімку на рік (кампанія вже завершена).

## Підключення реальних даних ЄДЕБО

### Результат розвідки (2026-07-22)

Агентська сесія без мережевого доступу не могла перевірити контракт даних
напряму. Наступна сесія мала повний доступ, але й вона **уперлась у той самий
брандмауер** — вихідний проксі агента блокує `vstup.edbo.gov.ua` (403 на
рівні політики), тож перевірка все одно робилась через
**`.github/workflows/diagnose-edbo.yml`** (GitHub Actions runner, повний
доступ в інтернет), результат читався напряму з логів джоба (не з
zip-артефакту — завантаження артефактів з Azure Blob Storage теж блокує
проксі агента).

Розширений `scripts/diagnose-edbo.mjs` дійшов до реальної форми пошуку і
з'ясував:

- Сайт — Next.js App Router SPA (`_next/static/chunks/*.js`, RSC-стрімінг
  `text/x-component`), **не** класичний REST/JSON API. Навігація
  "Конкурсні пропозиції" відкриває форму пошуку з полями «Освітній рівень»,
  «Основа вступу», **«Спеціальність (спеціалізація)»**, «Регіон», «Заклад
  освіти» і кнопкою «Пошук».
- **Реальний пошук захищений Cloudflare Turnstile.** Клік/запит до
  `POST https://vstup.edbo.gov.ua/offers` одразу тригерить
  `challenges.cloudflare.com` (видимий image-challenge, а не invisible-пас):
  у консолі видно типові бот-фінгерпринт-проби (`No available adapters`,
  WebGL-запити, приховані debug-рядки). Тобто звичайний headless Playwright
  розпізнається як бот і отримує капчу замість даних — сайт **явно і свідомо
  захищений від автоматизованого скрапінгу** саме на ендпоінті видачі
  рейтингових списків.

**Висновок:** писати скрипт, що обходить/розв'язує Cloudflare Turnstile, тут
свідомо не варто — це вже не "парсинг публічних даних", а обхід явного
анти-бот захисту державного сайту. `scripts/fetch-edbo.mjs` поки що **не
написано**. Життєздатні варіанти отримання реальних цифр:

1. **Ручний/напівручний імпорт.** Людина заходить на vstup.edbo.gov.ua у
   звичайному браузері (проходить Turnstile як звичайний користувач), робить
   пошук за спец. 061 «Журналістика» (бакалавр і магістр, усі ЗВО), і або
   експортує дані (якщо є кнопка експорту по кожній конкурсній пропозиції),
   або зберігає HTML/скріншот результатів — а окремий (некритичний до
   Cloudflare) скрипт лише парсить уже завантажений локальний файл в
   `data/*.json`.
2. **Офіційний відкритий датасет.** Перевірити, чи ЄДЕБО/МОН публікують ці ж
   рейтингові списки як відкриті дані на data.gov.ua чи через окремий
   офіційний API-ендпоінт без Cloudflare-гейту (за первинним оглядом такого
   не знайдено, але детально не перевірялось).
3. Якщо в майбутньому Cloudflare пропустить headless-запит без челенджу
   (наприклад, з іншого IP-діапазону чи after warm-up) — тоді `/offers`
   POST-запит із тілом форми (рівень, спеціальність, регіон) можна буде
   зловити так само через `diagnose-edbo.mjs` і побудувати на його основі
   `fetch-edbo.mjs`.

`.github/workflows/update-data.yml` лишається заготовкою (лише
`workflow_dispatch`, без `schedule`) — розкоментовувати й підключати
`js/data.js` до `fetch("data/...")` є сенс тільки після того, як з'явиться
робочий спосіб отримати дані одним із варіантів вище.

### Як перевіряли (для довідки)

0. **Розвідати реальний контракт даних** — **`.github/workflows/diagnose-edbo.yml`**
   (Actions → Diagnose EDBO data source → Run workflow), виконується на
   сервері GitHub із повним доступом в інтернет, відкриває сайт headless-
   браузером (`scripts/diagnose-edbo.mjs`, Playwright) і друкує в лог джоба
   (а не тільки в zip-артефакт `edbo-diagnostics`, який може бути недосяжний
   через egress-проксі агента) повний текст сторінки, посилання, кнопки,
   мережевий лог і знайдені `/api`-рядки в JS-бандлах.
1. **Отримати доступ до даних vstup.edbo.gov.ua** — див. висновок вище:
   реальний API захищений Cloudflare Turnstile, готового відкритого
   контракту для автоматизації поки що нема.
2. **Написати скрипт агрегації** — відкладено до появи життєздатного джерела
   (варіанти 1–3 вище). Формат вихідного знімка описаний нижче незмінний.
3. **Автоматизувати через GitHub Actions** — workflow-заготовка вже є, лишається
   неактивною (`workflow_dispatch` без `schedule`) до готовності кроку 2.

## Формат знімка даних

Кожен знімок (день для 2026-го, підсумок для 2021–2025) містить чотири
ранжовані списки — «усі заяви» і «лише пріоритет №1», окремо для кожного
рівня:

```js
{
  date: "2026-07-21",
  asOf: "2026-07-21T23:30:00+03:00",
  bachelor: [
    { rank: 1, id: "knu", name: "…", short: "КНУ", hue: 350, score: 178.1, applications: 646 },
    …
  ],
  master: [ … ],
  bachelorP1: [ … ],   // те саме, але тільки заяви з пріоритетом №1
  masterP1: [ … ],
  totalApplications: {
    bachelor: 3262, master: 1265,
    bachelorP1: 1480, masterP1: 540
  }
}
```

Знімки згруповані по роках у `DB.byYear[year] = { dates, snapshots }` (див.
`js/data.js`) — для 2026-го `dates` містить декілька днів кампанії, для
минулих років лише один (підсумковий).

## Ліцензія на дані

ЄДЕБО — державна база даних; рейтингові списки вступників публікуються
відкрито в межах вступної кампанії відповідно до законодавства про вступ до
закладів вищої освіти України. Перед публічним запуском варто звірити умови
використання цих даних (vstup.edbo.gov.ua) з юристом.
