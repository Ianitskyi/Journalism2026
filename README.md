# journalism2026

Рейтинг закладів вищої освіти України за кількістю заяв і середнім
конкурсним балом допущених до конкурсу на спеціальність **«Журналістика»**
(бакалавр і магістр).

Рейтинг рахується за даними ЄДЕБО й показує по кожному закладу освіти три
показники: усі подані заяви, допущені до конкурсу заяви та середній
конкурсний бал допущених. Бакалаврат і магістратура показані окремо.

🔗 Локально: відкрийте `index.html` подвійним кліком, або
```
python -m http.server 8080
```
і зайдіть на http://localhost:8080

## Статус: 2021–2025 реальні дані ЄДЕБО, 2026 поки демо

**2021–2025 — реальні дані**, зібрані `scripts/fetch-edbo-history.mjs` з
архівів `vstup<рік>.edbo.gov.ua` (детально — розділ нижче). Кількість
поданих і допущених заяв та середній бал допущених по кожному ЗВО —
справжні цифри вступної кампанії того року.

**Поточний 2026** — поки згенеровані тестові дані (`js/data.js`,
детерміновано на основі реальних назв ЗВО, але випадкові бали/заяви).
Живий `vstup.edbo.gov.ua` захищений
  Cloudflare Turnstile саме на видачі рейтингових списків — див. розділ
  нижче.

Перемикач бакалавр/магістр, перемикач року вступу (2021–2026), сортування
за середнім балом допущених або за кількістю всіх заяв, сторінка
кожного ЗВО з динамікою по роках і порівнянням з іншим ЗВО — усе працює як
задумано, з реальними цифрами там, де вони вже підключені.

## Підключення реальних даних ЄДЕБО

### Живий поточний рік (vstup.edbo.gov.ua, 2026) — потрібен ручний початковий список

Живий сайт — Next.js App Router. Реальний клієнтський контракт перевірено у
звичайній Chrome-сесії після людського проходження Cloudflare:

```text
Turnstile -> POST /api/verify-captcha {token}
apiPost("/vstup/speciality_offers/", {
  degreeId, specialityId, specializationId, eduBaseId, eduFormId,
  univId, studyProgramText, courseId, relevant, regionCode,
  textSearch, oc, isGrants
})
apiPost("/vstup/speciality_offers/univ", ті самі фільтри + univId)
```

`apiPost` не є звичайним JSON endpoint: дані гібридно шифруються і
передаються через Next.js Server Action `sendData`. Пошук виконується лише
після успішного `/api/verify-captcha`. Автоматизувати цю частину без людської
сесії означало б обходити антибот-захист, тому скрипти цього не роблять.

Після відкриття конкретної пропозиції ситуація інша: публічний
`GET /offer/<id>` працює без cookies. У Next.js-відповіді вже є готові
`stats.rqs_total` (усі подані заяви), `stats.rqs_allowed` (допущені до
конкурсу) і `stats.rqs_kv_avg` (середній конкурсний бал допущених). Тобто
повторно агрегувати всі індивідуальні бали для основного рейтингу не
потрібно.

#### Що потрібно зробити людині один раз

1. У Chrome відкрий `https://vstup.edbo.gov.ua/offers`, обери рівень,
   спеціальність **C7 Журналістика**, усі регіони й натисни **Пошук**.
2. Клікни назву закладу, потім біля кожної його пропозиції натисни
   **До списку вступників**.
3. На сторінці `/offer/<число>` натисни **F12** → **Console**, встав увесь
   вміст `scripts/edbo-capture.bookmarklet.js` і натисни Enter. Chrome збереже
   `.html` і `.json`; скрипт нічого не надсилає в мережу.
4. Повтори для всіх пропозицій бакалавра й магістра. Імпортуй отримані JSON
   (можна кілька через кому):

   ```bash
   node scripts/import-edbo-manual.mjs --date 2026-07-22 \
     --capture downloads/offer-1.json,downloads/offer-2.json
   ```

   Імпортер створить/оновить `data/2026-07-22.json`, `data/2026-index.json`
   і `data/2026-offers.json`. Повторний імпорт тієї самої пропозиції не
   дублює її. Індекс 2026 містить лише найсвіжішу дату: попередні денні
   знімки на сайті не показуються.
5. Коли `data/2026-offers.json` повний, наступні дні можна оновлювати без
   браузера командою `node scripts/fetch-edbo-current.mjs`. Вона читає лише
   публічні сторінки вже відомих id. Розклад GitHub Actions навмисно лишається
   закоментованим, доки повноту маніфесту і кілька стабільних запусків не
   перевірено.

### Архівні (завершені) роки vstup<рік>.edbo.gov.ua — 2021-2025 підключено

Архівні піддомени завершених кампаній **не захищені Cloudflare** (перевірено
`scripts/diagnose-edbo-history.mjs` — жоден з `vstup2018`…`vstup2025` не
тригерить Turnstile), тож автоматизація тут законна (це не обхід захисту —
захисту нема).

З коду `js/offers_search_form.js` (звичайний jQuery `$.ajax`, без токенів чи
підпису) з'ясовано точний контракт:

```
POST /offers-universities/  {qualification, education_base, speciality, region, education_form, course}
  → {universities: [{uid, un, ids: "id1,id2,...", n}, ...]}
POST /offers-list/  {ids: "id1,id2,..."}
  → {offers: [{uid, un, qid, qn, ssc, ssn, st: {c: {t, a, ka, km, kx, ...}}}, ...]}
```

`st.c.t` — усі подані заяви, `st.c.a` — допущені до конкурсу,
`st.c.ka` — середній конкурсний бал допущених по конкретній конкурсній
пропозиції. `scripts/fetch-edbo-history.mjs` агрегує всі
пропозиції спеціальності 061 «Журналістика» одного ЗВО в один рядок
(сумарні подані й допущені заяви; бал, зважений за кількістю допущених) і
записує знімок `data/<рік>-08-05.json`.

Важливий нюанс: **код спеціальності різний за роками**. 2018–2024
використовують стару числову класифікацію (`speciality=061`), а 2025 перейшов
на нову літерну (`speciality=C7`) — обидва означають те саме
"Журналістика". `scripts/fetch-edbo-history.mjs` вибирає код автоматично
(`specialityFor(year)`).

Для 19 ЗВО, що вже є у статичному списку демо-2026
(`BACHELOR_UNIS`/`MASTER_UNIS` у `js/data.js`), скрипт мапить EDBO `uid` на
той самий слаг (`knu`, `lnu`, `naukma`, …) — інакше сторінка ЗВО (графік
динаміки по роках) не змогла б з'єднати реальні роки з демо-2026 для того
самого закладу. Інші ЗВО (їх у реальних даних 60-70, значно більше за
демо-список) отримують id `edbo<uid>`.

**2018–2020 поки не підключено**: ті архіви мають іншу структуру
(`POST /offers-universities/` повертає HTML-помилку замість JSON) —
ймовірно інший ендпоінт чи технологія того покоління сайту. Не досліджено.

Основний рейтинг свідомо використовує **всі заяви незалежно від
пріоритету**. Одна людина може подати кілька заяв, тому це показник попиту,
а не кількість унікальних вступників чи фактично зарахованих студентів.

### Як оновити дані

Вручну: Actions → **Diagnose EDBO data source** → Run workflow (гілка з
цим кодом). Workflow виконує `scripts/fetch-edbo-history.mjs` для всіх
років і комітить `data/*.json` назад у гілку (`permissions: contents:
write`). Автоматичного розкладу (`schedule`) немає, бо архівні дані не
змінюються — оновлення потрібне лише коли підключаються нові роки/дані.

## Формат знімка даних

Кожен знімок (день для 2026-го, підсумок для 2021–2025) містить основні
ранжовані списки для бакалаврату й магістратури. Поля `*P1` лишаються в
контракті для зворотної сумісності імпортера, але головний рейтинг їх не
використовує:

```js
{
  date: "2026-07-22",
  asOf: "2026-07-22T23:30:00+03:00",
  bachelor: [
    { rank: 1, id: "knu", name: "…", short: "КНУ", hue: 350,
      score: 178.1, applications: 646, admitted: 521 },
    …
  ],
  master: [ … ],
  bachelorP1: [ … ],   // те саме, але тільки заяви з пріоритетом №1
  masterP1: [ … ],
  totalApplications: {
    bachelor: 3262, master: 1265,
    bachelorP1: 1480, masterP1: 540
  },
  totalAdmitted: {
    bachelor: 2751, master: 1034
  }
}
```

`applications` — усі подані заяви, `admitted` — допущені до конкурсу,
`score` — середній конкурсний бал допущених. Якщо один ЗВО має кілька
конкурсних пропозицій, `score` агрегується як зважене середнє з вагою
`admitted`, а не `applications`.

Знімки згруповані по роках у `DB.byYear[year] = { dates, snapshots }` (див.
`js/data.js`). Для 2026-го завантажується лише найсвіжіший доступний знімок;
для минулих років — по одному підсумковому.

## Ліцензія на дані

ЄДЕБО — державна база даних; рейтингові списки вступників публікуються
відкрито в межах вступної кампанії відповідно до законодавства про вступ до
закладів вищої освіти України. Перед публічним запуском варто звірити умови
використання цих даних (vstup.edbo.gov.ua) з юристом.
